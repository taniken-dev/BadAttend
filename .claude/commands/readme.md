# /readme

最新のgitログと現在のREADMEを照らし合わせ、READMEを最新の状態に更新します。

## 手順

1. `git log --oneline -30` で直近30件のコミット履歴を確認する
2. 現在の `README.md` を読み込む
3. コミット履歴と照らし合わせ、READMEに反映されていない変更・追加機能・削除機能を特定する
4. 以下の方針でREADMEを更新する：
   - 実装済みの機能・仕様のみを記載する（未実装・将来予定は書かない）
   - 削除された機能は記述を取り除く
   - 新機能はそれに対応するセクションに追記する（新セクションが必要なら作る）
   - 技術スタック・環境変数は実際のコードと一致させる
   - 運営ルール・制約は最新のロジックに合わせる
   - 文体・フォーマットは既存のREADMEスタイルを踏襲する
5. `README.md` を上書き保存する
6. 変更内容の要点を箇条書きで報告する

## 注意事項

- コードを読む必要がある場合は該当ファイルを Read して確認してから書く
- ユーザーへの確認なしで更新してよい（スキル発動が承認とみなす）
- コミットは行わない（更新のみ）
